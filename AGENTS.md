# JTBC News Camera Hub - Codex Instructions

## 프로젝트 개요
이 프로젝트는 Next.js 15 App Router + Supabase + Vercel 기반의 내부 포털이다.
주요 기능은 로그인, 근무표, 휴가신청, 베스트리포트 평가, 팀장 기능, 관리자 기능, 홈 뉴스 브리핑이다.

## 기본 원칙
- 기존 기능을 깨지 않는 최소 변경을 우선한다.
- Next.js 15 App Router 구조를 유지한다.
- Supabase Auth, RLS, profiles.role, approved 흐름을 임의로 바꾸지 않는다.
- 로그인, 권한, 근무표, 휴가신청, 베스트리포트, 관리자, 팀장 기능은 회귀 테스트 대상으로 본다.
- 홈 화면 초기 로딩을 무겁게 만들지 않는다.
- Supabase 호출 수와 egress 증가를 항상 고려한다.
- 모바일 화면을 우선 확인한다.

## 금지
- .env.local 내용을 커밋하지 않는다.
- SUPABASE_SERVICE_ROLE_KEY를 클라이언트 코드에 노출하지 않는다.
- middleware와 AuthGate는 꼭 필요한 경우가 아니면 건드리지 않는다.
- 홈 화면에서 불필요한 전체 월 근무표 fetch를 추가하지 않는다.
- 대규모 리팩터링을 임의로 하지 않는다.
- RLS를 완화해서 문제를 해결하지 않는다.

## 작업 방식
- 수정 전 관련 파일과 데이터 흐름을 먼저 파악한다.
- DB 변경이 필요하면 schema.sql 수정 내용과 실제 Supabase SQL Editor에 적용할 SQL을 구분해 설명한다.
- UI 변경은 모바일, 태블릿, 데스크톱 영향을 함께 검토한다.
- 기능 추가는 1차 구현 범위를 작게 자른다.
