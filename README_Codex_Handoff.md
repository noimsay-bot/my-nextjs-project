## 3. 근무표 기능 (최우선)
### 3-1. 생성 범위
- 월별 근무표 자동 작성
- 그달 마지막 날이 주중에 끝나면 그 주 일요일까지 계속 작성
- 다음 달 근무표는 그 다음 월요일부터 이어서 작성

### 3-2. 평일 규칙
- 조근 2명
- 연장 4명
- 석근 3명
- 야근 1명
- 제크 1~2명
- 휴가 칸 표시
- 평일에는 국회/청사/청와대 칸 없음

### 3-3. 주말 규칙
- 조근 1명
- 일반근무 2명
- 뉴스대기 1명
- 야근 1명
- 제크 칸 없음
- 주말/휴일에는 국회 / 청사 / 청와대 수동 입력칸 표시
- 주말에는 휴가 칸 없음

### 3-4. 순번 규칙
- 조근: 일 단위
- 석근: 일 단위
- 야근:
  - 월~목 야근 순번
  - 금요일 야근 순번
  - 토요일 야근 순번
  - 일요일 야근 순번
- 제크: 일 단위
- 휴일근무: 순번 기반
- 평일휴일근무: 순번 기반
- 연장:
  - 주 단위
  - 한 주에 뽑힌 4명이 월요일~금요일 동일
  - 다음 순번은 다음 주 월요일부터 적용

### 3-5. 휴일근무 규칙
- 휴일근무 순번에서 4명을 가져옴
- 1번째 순번: 조근
- 2,3번째: 일반근무
- 4번째: 뉴스대기
- 야근은 별도의 요일별 야근 순번 사용

### 3-6. 순번 입력 페이지
- 근무 종류별 카테고리 분리
- 각 카테고리마다 1번~30번 입력칸
- 각 카테고리별 이름 목록 입력 가능
- 실시간 미리보기
- 특정 인원 오프/활성화 가능
- 수정 즉시 미리보기에 반영

### 3-7. 오프 기능
- 특정 인원을 오프시키면 자동배치에서 제외
- 다시 활성화하면 순번에 다시 반영
- 입력값 자체는 유지

### 3-8. 충돌 처리
- 야근 다음날 근무가 있으면 충돌 표시
- 자동 재배치 전에 원본 사본 저장
- 자동 재배치 기능 유지

### 3-9. 수동 수정 UX
- 날짜 카드마다 수정 버튼
- 수정 누르면 각 근무 유형 옆에 + 버튼 생성
- 각 이름 옆에 - 버튼 생성
- 이름은 클릭 선택 또는 드래그로 다른 칸 이동 가능
- 날짜 카드에 추가 수동 칸 생성 기능 유지

### 3-10. 휴가 입력
- 평일 휴가 칸은 수정 모드에서 쉼표로 한 번에 입력 가능
- 예: 정철원, 홍길동, 김민수
- 입력하면 이름이 각각 분리된 객체형 칩으로 표시

### 3-11. 표시 UX
- 각 근무유형의 이름은 세로로만 늘어지지 말고 두 줄 중심으로 배치
- 로그인한 사람 이름을 입력하고 “내 근무 보기”를 누르면 해당 이름이 하늘색으로 강조

## 5. 영상평가 제출 기능
### 5-1. 제출 화면
- 로그인 후 제출 가능
- 폼 종류 선택 영역은 따로 두지 않음
- “폼 추가” 버튼으로 카드 추가
- 카드 안에서 폼 종류 선택
- 최대 3개까지 제출 가능

### 5-2. 폼 종류
- 일반리포트
- 기획리포트
- 인터뷰리포트
- LIVE(기자연결, 현장연결, 모바일)

### 5-3. 각 폼 입력값
- 제목
- 링크
- 날짜
- 코멘트

### 5-4. 제출
- 제출 시 로그인한 사람 이름으로 저장
- 같은 사람이 다시 제출하면 최신본으로 갱신 가능하게 설계
- 제출 데이터는 평가페이지로 연결

## 6. 평가페이지
### 6-1. 접근 권한
- reviewer / team_lead만 접근 가능

### 6-2. 구조
- PC: 왼쪽 이름 탭 / 오른쪽 평가표
- 모바일: 상단 가로 스크롤 탭
- 이름을 누르면 해당 제출자의 평가표 표시

### 6-3. 채점 방식
- 각 폼 종류별 세부 평가 기준 존재
- 각 세부 항목 옆에 “충족” 버튼
- 누르면 해당 배점 합산
- 다시 누르면 해제 가능

### 6-4. 추가 가점
- 0~5점
- 1점 이상이면 반드시 추가가점의견 입력
- 없으면 평가 완료 처리 불가

### 6-5. 상태
- 평가 대기
- 평가 진행 중
- 평가 완료

### 6-6. 평가 완료
- 버튼으로 완료 처리
- 완료 조건 검증 필요

## 7. 폼 종류별 채점표
코드에서 템플릿화해서 관리

### 일반리포트
1. 기사 주제와의 정합성
- 영상이 기사 핵심 내용과 직관적으로 매칭되는가 (3)
- 전개 흐름에 맞는 컷 구성과 연결이 자연스러운가 (3)

2. 기술적 완성도 및 기본기
- 수평·수직·사이즈·헤드룸 등이 안정적인가 (2)
- 화이트밸런스·노출·초점이 정확하고 현장 오디오가 명료한가 (3)

3. 기본 영상 확보 및 현장 커버리지
- 기사 구성에 필요한 사이즈별 영상이 충분한가 (3)
- 인서트 및 상황 설명 화면을 적절히 확보했는가 (3)

4. 현장 활용 및 대응력
- 제한된 환경 속에서 최적의 앵글과 배경을 선택했는가 (3)

5. 가점
- 모바일라이브 또는 콘텐츠 제작에 선제 대응하였는가 (1)
- 현장 돌발 상황에 유연하게 대응했는가 (1)

### 기획리포트
1. 메시지 전달력 및 스토리텔링
- 영상이 기획 의도와 핵심 메시지를 명확히 전달하는가 (3)
- 화면 호흡과 흐름이 자연스럽고 몰입도가 있는가 (3)

2. 영상 표현력
- 심도·프레이밍·카메라 무빙이 의도적으로 설계되었는가 (3)
- 인물 심리와 현장 상황을 고려한 프레임 구성인가 (3)

3. 빛, 공간 해석 능력
- 자연광·조명을 전략적으로 활용했는가 (3)
- 공간을 입체적으로 활용해 주제를 강화했는가 (3)

4. 오디오·현장감 완성도
- 인터뷰이 오디오가 선명하고 현장의 음향을 의도적으로 컨트롤하였는가 (2)

5. 가점
- 동일 사안 대비 독창적 영상 접근이 있는가 (1)
- 확장성(추가 콘텐츠 제작)을 고려하여 구성하였나 (1)

### 인터뷰리포트
1. 구도 안정성
- 주제와 어울리는 적절한 사이즈와 배경을 선정했는가 (5)

2. 인물 부각 및 배경 선택
- 조명·자연광을 활용해 인물을 효과적으로 부각했는가 (5)

3. 오디오 완성도
- 목소리를 노이즈 없이 명료하게 수음했는가 (5)

4. 내용 보완 화면 확보
- 제스처·반응 컷 등 인서트를 충분히 확보했는가 (5)

5. 가점
- 로드 인터뷰·돌발 상황에서 구도 유지와 피벗팅이 매끄러운가 (1)
- 다인 인터뷰 시 균형이 유지되는가 (1)

### LIVE
1. 시각적 현장성
- 보도 주제를 가장 잘 설명하는 배경을 선택했는가 (4)
- 주·야간 환경에서 기자(혹은 주요 피사체)가 선명히 표현되는가 (3)

2. 기술 안정성
- 화면 수평·구도가 안정적인가 (3)
- 송출 신호·화질이 안정적인가 (3)

3. 오디오 명료성
- 취재기자 또는 출연자의 음성과 현장음이 혼선 없이 전달되는가 (3)

4. 카메라 운용능력
- 워크앤토크·이동 동선에 맞는 매끄러운 무빙인가 (4)

5. 가점
- 라이브 이후 클립 재가공이 가능한 소스를 확보했는가 (1)
- 현장 돌발 상황에 즉각 대응했는가 (1)

## 8. 팀장페이지 / 보고서
### 8-1. 접근 권한
- team_lead만 접근
- admin은 접근 불가

### 8-2. 기능
- 평가자 지정
- 평가자 초기화
- 팀장 보고서 열람

### 8-3. 보고서 구조
- 1분기 / 2분기 / 3분기 / 4분기 / 연도별 페이지

### 8-4. 분기별 페이지
- 각 인원당
  - 평가자별 채점표
  - 각 리포트별 점수
  - 평가자별 세부 채점 항목 상세 보기
  - 리포트별 평균 점수
  - 가장 높은 점수를 받은 리포트 강조 표시

### 8-5. 연도별 페이지
- 각 인원당
  - 분기별 리포트 중 가장 높은 점수의 리포트만 추출
  - 그 점수만 합산해 연도 점수 표시

## 10. Codex에 시킬 첫 작업
1. `schedule-integrated-v10.html`을 기준으로 React/Next.js 구조로 근무표 기능부터 옮긴다.
2. 사용자가 제공한 순번표를 기본값 seed 데이터로 반영한다.
3. 근무표 기능이 안정화된 뒤 영상평가/평가/팀장/관리자 기능을 붙인다.
4. 파일 단위 변경사항과 실행 방법을 매 단계 설명한다.
