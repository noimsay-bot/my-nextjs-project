# code_review.md

## 리뷰 목표
이 프로젝트의 리뷰는 기능 추가보다 먼저 아래를 본다.
- 보안
- 권한
- 빌드 안정성
- 기존 기능 회귀
- 모바일 UI 안정성

## 반드시 체크할 항목
1. 권한 우회 가능성이 없는가
2. approved 체크가 빠지지 않았는가
3. 서버에서 막아야 할 것을 클라이언트에만 맡기지 않았는가
4. 타입 에러를 억지 캐스팅으로 숨기지 않았는가
5. import 누락이나 빌드 에러 가능성이 없는가
6. 기존 페이지 동작을 깨뜨리지 않는가
7. 모바일에서 레이아웃이 깨지지 않는가
8. Supabase 스키마 변경 시 SQL과 RLS 검토가 포함되었는가
9. 불필요한 리팩터링이 섞이지 않았는가
10. 변경 파일 수가 과도하게 넓지 않은가

## 리뷰 코멘트 방식
- 문제가 있으면 파일 경로와 원인을 먼저 적는다.
- 왜 위험한지 짧고 명확하게 적는다.
- 가능하면 최소 수정 대안을 같이 적는다.

## 이 프로젝트에서 특히 민감한 영역
- middleware
- auth gate
- portal 레이아웃
- admin / desk / team-lead 권한 페이지
- schedule 관련 로직
- home 뉴스 브리핑 로직
- Supabase RLS / role / approved 관련 SQL
