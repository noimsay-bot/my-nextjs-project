# app/AGENTS.md

이 폴더에서는 라우팅 구조와 페이지 조합 방식을 우선 존중한다.

## 규칙
- page.tsx는 페이지 조합 중심으로 유지한다.
- 무거운 UI 로직은 components로 이동한다.
- 서버/클라이언트 경계를 함부로 바꾸지 않는다.
- layout.tsx 수정은 영향 범위가 크므로 꼭 필요한 경우만 한다.
- 공개 페이지와 portal 라우트 그룹 구조를 함부로 흔들지 않는다.
- 페이지 파일 하나에서 너무 많은 상태와 로직을 처리하지 않는다.
- route, action, page의 역할을 분리한다.
